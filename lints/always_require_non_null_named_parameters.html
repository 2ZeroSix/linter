always_require_non_null_named_parameters

Group: style

Maturity: stable

View the Project on GitHub dart-lang/linter

  • See the Style Guide
  • List of Lint Rules

DO specify @required on named parameter without default value on which an assert(param != null) is done.

GOOD:

m1({@required a}) {
  assert(a != null);
}

m2({a: 1}) {
  assert(a != null);
}

BAD:

m1({a}) {
  assert(a != null);
}

NOTE: Only asserts at the start of the bodies will be taken into account.
